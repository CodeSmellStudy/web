Exploring The Existence and Impacts of Code Smells on Android Mobile Projects

Abstract

A code smell is an implementation that often reflects a deeper problem in the program. Due to the need for rapid delivery or the lack of experience of developers, code smells are commonly introduced in software development. Numerous prior studies have presented that code smells could negatively affect the development and maintenance of software projects. In recent years, with continuous and rapid growth, hundreds and thousands of mobile applications have been implemented and used in our daily lives. However, there is little work investigating the impacts of code smells on mobile applications. To bridge this gap, we explore whether and how code smells influence mobile projects. Through our analysis of twenty Android mobile projects, we have presented that: 1) A large portion of files in mobile projects encounter code smells; 2) The files involved in code smells are often more bug-prone; 3) A file with more code smells becomes more bug-prone; 4) The \emph{Unutilized Abstraction} smell has the largest impact scope, and the \emph{Long Statement} smell contributes the most to the bug-proneness of files in mobile projects.
